# Context Engineering Rules

Put the right information in the model for the current phase. Do not maximize
context volume.

## Phases

1. **Intake** — classify lane; read `docs/FEATURE_INTAKE.md` and relevant product docs.
2. **Planning** — pick smallest safe approach and proof.
3. **Implementation** — stay inside the story file set.
4. **Validation** — run stated proof; do not claim unproven completion.
5. **Trace** — leave evidence for the next agent.

Skip unrelated history and open files that do not affect the selected story.
